# Epic PRD: Mega-Quest MVP — 모바일 기반 온보딩 통합 에이전트

---

## 1. Epic Name

**Mega-Quest MVP: 게이미피케이션 기반 신입사원 온보딩 웹앱**

---

## 2. Goal

### 문제 (Problem)
신입사원은 입사 초기에 회사 내 정보(인사제도, 시스템 권한, 업무 절차 등)를 파악하는 데 많은 시간과 에너지를 소모한다. 공채/수시 등 채용 형태에 따라 온보딩 경험의 질이 불균등하며, 체계적인 가이드 없이 개인이 정보를 직접 탐색해야 하는 경우가 많다. 또한 일반적인 온보딩 자료는 단방향 전달 방식이라 신입사원의 능동적 참여를 이끌어내기 어렵다.

### 해결책 (Solution)
Mega-Quest는 퀘스트 시스템과 가상 마스코트("트리니티 펫")를 활용한 모바일 웹앱으로, 신입사원이 온보딩 미션을 게임처럼 수행하며 자연스럽게 회사 정보를 습득하도록 유도한다. 채용 형태에 관계없이 동일한 표준화된 온보딩 경험을 제공하며, 포인트 보상과 펫 성장이라는 즉각적 피드백으로 참여 동기를 부여한다.

### 기대 효과 (Impact)
- 신입사원의 온보딩 정보 탐색 시간 단축
- 온보딩 미션 완료율 향상
- 신입사원 만족도 및 초기 몰입도 증대
- 채용 형태별 온보딩 품질 편차 해소

## 3. User Personas

### 페르소나 1: 신입사원 (Primary User)
- **이름:** 김신입 (26세)
- **상황:** 대기업 공채로 입사한 지 3일차. 사내 시스템이 생소하고 누구에게 무엇을 물어봐야 할지 막막함.
- **목표:** 빠르게 회사에 적응하고, 필요한 정보를 스스로 찾아 업무를 시작하고 싶음.
- **Pain Point:** 방대한 온보딩 자료, 선배 눈치 보기, 중복 질문에 대한 부담.
- **디바이스:** 주로 모바일(스마트폰)로 접속.

### 페르소나 2: HR 담당자 (Secondary / Demo Evaluator)
- **이름:** 박인사 (34세)
- **상황:** 해커톤 심사자로서 이 솔루션의 사업성과 UX를 평가.
- **목표:** 신입사원 온보딩 효율화 가능성과 확장 가능성 확인.
- **관심사:** 데모 완성도, 실제 도입 시 ROI, 유지보수 용이성.

## 4. High-Level User Journeys

### Journey 1: 앱 진입 및 온보딩
1. 신입사원이 모바일로 Mega-Quest에 접속 → 온보딩 화면(`/onboarding`) 표시.
2. 서비스 소개(퀘스트 수행, 펫 성장, 포인트 적립) 확인 후 "시작하기" 클릭.
3. 로그인 화면(`/login`)에서 "Google로 계속하기 (회사 계정)" 클릭 (Mock — 실제 OAuth 없음).
4. 메인 대시보드(`/`)로 진입.

### Journey 2: 온보딩 미션 탐색 및 수행
1. 메인 대시보드에서 트리니티 펫(현재 1단계: 알)과 포인트 진행도 확인.
2. '퀘스트' 탭으로 이동하여 전체 미션 로드맵 확인 (N/28 완료 배너).
3. 카테고리 탭(7개)에서 원하는 카테고리 선택.
4. 퀘스트 선택 → 미션 수행:
   - '회의실 QR 체크인' 퀘스트: QR 스캔 시뮬레이터 UI → "스캔하기" → 2초 로딩 → 완료 모달.
   - 기타 퀘스트: "달성하셨나요?" 확인 플로우 → "확인" → 1초 로딩 → "확인되었습니다!" → 1초 → 완료 모달.
5. 완료 모달에서 "다음 퀘스트 하기" 또는 "홈으로" 선택.
6. 메인 화면으로 돌아와 펫 성장 상태 및 포인트 확인.

### Journey 3: 펫 성장 확인
1. 퀘스트를 완료할 때마다 포인트 누적.
2. 포인트 임계치 도달 시 펫이 다음 단계로 진화:
   - Stage 1 (0~9P): 🥚 알 (Egg)
   - Stage 2 (10~14P): 🐣 아기 트리니티
   - Stage 3 (15P+): ✨ 메가 트리니티
3. 프로그레스 바로 다음 단계까지 남은 포인트 확인.

## 5. Business Requirements

### 기능 요구사항 (Functional Requirements)

**온보딩 플로우 (Onboarding)**
- `/onboarding` 페이지: 블루 그라데이션 배경, 서비스 소개 3개 카드(퀘스트 수행/펫 성장/명예 칭호), "시작하기" 버튼
- 매 실행 시 항상 표시 (localStorage 기반 skip 없음)
- `/onboarding`, `/login` 경로에서 하단 BottomNav 미표시

**로그인 플로우 (Login)**
- `/login` 페이지: Google 로그인 버튼(Mock — UI 전용, 실제 OAuth 없음)
- Google 4색 로고 SVG + "Google로 계속하기 (회사 계정)" 버튼
- 클릭 시 즉시 홈 대시보드(`/`)로 이동

**메인 대시보드 (Home)**
- 사용자 인사말 + 이름 표시, 명예 칭호 배지
- 트리니티 펫을 포인트에 따라 3단계로 시각화 (이모지 전용: 🥚→🐣→✨)
- 현재 포인트 및 다음 단계까지의 프로그레스 바 표시
- '오늘의 일일 퀘스트' 바로가기 버튼 (전체 완료 시 "오늘의 미션 완료! 🎉"로 변경)
- 미완료 일일 퀘스트 목록 표시

**퀘스트 로드맵 (Quest List)**
- 카테고리 탭 7개 — 온보딩 여정 순서대로 정렬:
  [입사 전 준비], [첫 출근], [필수 교육], [조직 이해], [HR 초보자], [직군별 전직], [일일/월간]
- 퀘스트 카드: 제목, 보상 포인트, 상태 배지(도전 가능/완료/잠김), 상태 아이콘(🎯/✅/🔒)
- 전체 진행률 배너: 상단에 `N/28 완료` 표시
- 기본 탭: 미완료 퀘스트가 있는 첫 번째 카테고리로 자동 포커스
- 잠긴 퀘스트: opacity-50, 클릭 비활성, pointer-events-none
- 탭 스크롤: 우측 fade gradient 표시

**미션 수행 — QR 퀘스트 (`meeting-room-qr`)**
- 카메라 뷰를 모사한 QR 스캔 UI (검정 배경, 코너 마커 4개, 스캔 라인)
- '스캔하기' 버튼 클릭 → 2초 로딩 → `completeQuest` 호출 → 완료 모달 표시

**미션 수행 — 일반 퀘스트 (QR 외 전체)**
- "달성하셨나요?" 확인 플로우:
  - Step 1: 퀘스트 정보(아이콘/제목/설명/보상) + "확인" 버튼
  - Step 2: "확인" 클릭 → 스피너 + "확인하는 중..." (1초)
  - Step 3: 체크 아이콘 + "확인되었습니다! ✅" (1초)
  - Step 4: `completeQuest` 호출 → 완료 모달 표시
- 로딩/확인 중 버튼 비활성화

**완료 모달 (CompletionModal)**
- 체크 아이콘 + "미션 완료!" + 퀘스트 제목 + 획득 포인트
- "다음 퀘스트 하기" 버튼 (→ `/quests`)
- "홈으로" 버튼 (→ `/`)

**퀘스트 상태별 화면 분기**
- `completed` 상태 퀘스트 재방문: "완료한 퀘스트예요!" 안내 + 획득 포인트 표시 + "홈으로" 버튼
- `locked` 상태 퀘스트 직접 접근: 🔒 잠금 안내 + 카테고리별 힌트 + "퀘스트 목록으로" 버튼
- 존재하지 않는 퀘스트 ID 접근: "퀘스트를 찾을 수 없습니다" 안내

**전역 상태 관리**
- Zustand 5 + persist middleware (`mega-quest-storage`, version: 4)
- `completeQuest(questId)`: `available` 상태 퀘스트만 처리, 포인트 + petStage 자동 갱신
- `resetGame()`: 초기 상태로 리셋
- localStorage 기반 영속성, version 변경 시 자동 초기화

**하단 네비게이션 (BottomNav)**
- 홈(🏠) / 퀘스트(📋) 2개 탭
- `/onboarding`, `/login` 경로에서 숨김 (NavWrapper 컴포넌트)
- Safe area inset 지원 (`env(safe-area-inset-bottom)`)

### 비기능 요구사항 (Non-Functional Requirements)
- **Mobile-First:** 모든 UI는 모바일 뷰포트(375px 이상) 기준으로 최적화
- **반응형 디자인:** 태블릿 및 데스크톱에서도 정상 동작 (max-w-md 제한)
- **성능:** 초기 로드 시간 3초 이내 (모바일 기준)
- **접근성:** 버튼에 aria-label, 프로그레스바에 role="progressbar", 모달에 role="dialog" + aria-modal
- **데이터 영속성:** localStorage (Zustand persist), mock 데이터 사용; 실제 DB 연동 불필요
- **보안:** 인증/인가는 Mock UI 전용 — 실제 OAuth/세션/토큰 처리 없음

## 9. Business Value

**평가: High**

해커톤 맥락에서 이 에픽은 다음 이유로 높은 비즈니스 가치를 가진다:
- **차별화된 데모 임팩트:** 온보딩→로그인→퀘스트→펫 성장의 완결된 플로우 + 게이미피케이션 + 모바일 최적화 조합은 심사자에게 강한 인상을 남길 수 있음.
- **실제 문제 해결:** 신입사원 온보딩 비효율은 대부분의 기업이 공감하는 실존 문제로, 솔루션의 필요성이 분명함.
- **확장 가능성:** MVP를 기반으로 실제 HR 시스템 연동, AI Q&A, 관리자 기능 등으로 발전 가능.
- **빠른 구현 가능성:** Mock 데이터 + 프론트엔드 중심 구조로 해커톤 시간 내 완성 가능.
